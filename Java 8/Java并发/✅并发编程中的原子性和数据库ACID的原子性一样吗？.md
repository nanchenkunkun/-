**不一样。原子性在并发编程中，和在数据库中两种不同的概念。**

在数据库中，事务的ACID中原子性指的是"要么都执行要么都回滚"。

在并发编程中，原子性指的是"操作不可拆分、不被中断"。所以在并发编程中，我们要保证原子性指的就是一段代码需要不可拆分，不被中断。

比如经典的i++问题，就是原子性的一个很好地体现。
